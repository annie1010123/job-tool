# JobPilot — User Stories

## User Story 清單

### US-001：每日自動推薦職缺

**身為**積極求職者，**我想要**每天自動收到匹配的職缺推薦，**以便**不用每天花 1 小時逛 104。

**驗收標準：**
- [ ] 每天 08:00 收到 Email 推薦
- [ ] Dashboard 同時顯示推薦清單
- [ ] 推薦基於 embedding 語意匹配（0.5×意圖 + 0.5×關鍵字 × boost）
- [ ] 推薦職缺有相符分數、公司名、薪資、徵才積極度

**對應 Phase：** Phase 1（W1-2）— 已完成（v1 既有功能）

---

### US-002：一鍵收藏 + 統一管理職缺

**身為**積極求職者，**我想要**一鍵收藏感興趣的職缺，**以便**統一管理不同平台看到的機會。

**驗收標準：**
- [ ] 推薦列表每個職缺有「收藏」按鈕
- [ ] 手動貼 URL 也能收藏（自動解析公司名+職稱）
- [ ] 可按平台篩選（104 / LinkedIn / Cake / Yourator / 其他）
- [ ] 可按公司類型篩選（新創 / 大公司 / 傳產）
- [ ] URL 為選填（手動新增時可以不填）

**對應 Phase：** Phase 1（W1-2）

---

### US-003：AI 生成客製化推薦信

**身為**積極求職者，**我想要**AI 根據我的履歷和 JD 生成推薦信，**以便**每次投遞省下 40 分鐘。

**驗收標準：**
- [ ] 選語氣（正式 / 活潑 / 簡潔）
- [ ] 30 秒內生成 300-500 字推薦信
- [ ] 結果可編輯 + 一鍵複製
- [ ] 可重新生成
- [ ] 根據 WorkExperience 挑選最相關經歷（STAR 法則）
- [ ] 履歷未上傳時提示先完成

**對應 Phase：** Phase 2（W3-4）

---

### US-004：面試後 AI 復盤

**身為**積極求職者，**我想要**面試後上傳錄音讓 AI 幫我復盤，**以便**知道哪些題目答得好、哪些需要改進。

**驗收標準：**
- [ ] 上傳錄音（mp3/m4a/wav, max 50MB）→ Whisper 轉文字
- [ ] AI 萃取每題 Q&A
- [ ] 每題標注 good / ok / needs_improvement + 改善建議
- [ ] 生成整體反饋（摘要 + 3 個改善重點）
- [ ] 也可以手動填寫（不想錄音時的 fallback）
- [ ] 手動填寫 1 分鐘內可完成

**對應 Phase：** Phase 3（W5-7）

---

### US-005：AI 面試準備越來越精準

**身為**積極求職者，**我想要**AI 面試準備越來越精準，**以便**第 5 次面試的準備品質比第 1 次好。

**驗收標準：**
- [ ] 復盤的 Q&A 自動寫入 QuestionBank
- [ ] 下次面試準備時，AI 從題庫挑選相關題目
- [ ] 有歷史資料的題目標注「⚡ 根據你上次面試經驗延伸」
- [ ] 附上上次的表現評估和改善建議
- [ ] 沒有歷史資料時退化為純 JD 出題（backward compatible）
- [ ] 底部引導：「完成這次面試的復盤，下次 AI 會更了解你」

**對應 Phase：** Phase 3（W5-7）

---

### US-006：不用註冊就能體驗核心功能

**身為**作品集 Reviewer，**我想要**不用註冊就能體驗核心功能，**以便**3 分鐘內判斷候選人的能力。

**驗收標準：**
- [ ] Landing Page 不需登入即可訪問
- [ ] 貼 JD 文字 → 30 秒內生成推薦信草稿
- [ ] Rate limit：每 IP 每小時 5 次
- [ ] 結果下方有 CTA：「想儲存並管理更多？免費註冊」
- [ ] 頁面載入 < 2 秒

**對應 Phase：** Phase 0（W0）

---

### US-007：全流程投遞追蹤

**身為**積極求職者，**我想要**在同一個看板追蹤所有投遞進度，**以便**不會搞混 15 間公司的面試狀態。

**驗收標準：**
- [ ] Kanban 看板 5 欄：未投遞 → 投遞中 → 面試中 → 二面 → 結果
- [ ] 從收藏區投遞時自動建立追蹤 + 移到「已投遞」
- [ ] 可拖拉移動狀態
- [ ] 每張卡片顯示公司名、職稱、平台標籤、日期

**對應 Phase：** Phase 1（W1-2）— 部分已完成（v1 Kanban + v2 收藏串接）
